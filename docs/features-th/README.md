# Neatly Hotel - เอกสารคุณสมบัติและฟีเจอร์

ไดเรกทอรีนี้มีเอกสารทางเทคนิคที่ครอบคลุมสำหรับฟีเจอร์และโมดูลหลักทั้งหมดในระบบจัดการโรงแรม Neatly

---

## 📚 สารบัญเอกสาร

### ฟีเจอร์หลักของระบบ

1. **[ระบบยืนยันตัวตนและการอนุญาต](./authentication.md)**
   - การลงทะเบียนและเข้าสู่ระบบผู้ใช้
   - การควบคุมการเข้าถึงตามบทบาท (ลูกค้า/แอดมิน)
   - รองรับผู้ใช้แบบไม่ต้องลงทะเบียน (Guest)
   - การจัดการเซสชันและการเชื่อมโยง
   - การยืนยันอีเมลและรีเซ็ตรหัสผ่าน

2. **[ระบบแชทบอทด้วย AI (พร้อมแชทสดและระบบตั๋วสนับสนุน)](./chatbot.md)**
   - ระบบตอบกลับแบบผสมผสาน (การจับคู่แบบตรงตัว + vector search + การจำแนกเจตนา)
   - การตอบกลับหลายรูปแบบ (ข้อความ, ตัวเลือก, การ์ดห้องพัก)
   - OpenAI embeddings + Supabase pgvector
   - ติดตามบริบทการสนทนา
   - การส่งต่อไปยังแชทสดและระบบตั๋วสนับสนุน
   - การส่งข้อความระหว่างแอดมินและลูกค้าแบบเรียลไทม์
   - การจัดการวงจรชีวิตของตั๋ว (เปิด → กำลังดำเนินการ → ปิด)

3. **[ระบบจองและชำระเงิน](./booking-system.md)**
   - ตรวจสอบห้องว่างและสร้างการจอง
   - ราคาแบบไดนามิก (ราคาฐาน, โปรโมชัน, คำขอพิเศษ, โค้ดส่วนลด)
   - นโยบายยกเลิกภายใน 24 ชั่วโมงและการจองหลายห้อง
   - ชำระเงินด้วยบัตรเครดิตผ่าน Stripe (โหมดทดสอบพร้อมบัตรทดสอบ)
   - รองรับการชำระเงินสด (ชำระเมื่อเช็คอิน)
   - การตรวจสอบ ประมวลผล และคืนเงิน
   - ยืนยันการจองอัตโนมัติเมื่อชำระเงินสำเร็จ
   - ติดตามสถานะการชำระเงิน (รอดำเนินการ → เสร็จสิ้น → คืนเงิน)

4. **[ระบบจัดการห้องพักและประเภทห้องพัก](./room-management.md)**
   - โครงสร้างสองชั้น (ประเภทห้องพัก + ห้องพักจริง)
   - การจัดการรูปภาพ (รูปหลัก + แกลเลอรี)
   - ราคาโปรโมชัน
   - ติดตามสถานะห้องพัก
   - การจัดการสิ่งอำนวยความสะดวก
   - การปรับแต่งภาพด้วย jimp

5. **[ระบบโปรโมชันและส่วนลด](./promotion-system.md)**
   - สร้างและจัดการโค้ดส่วนลด
   - ส่วนลดแบบจำนวนเงินคงที่และเปอร์เซ็นต์
   - ติดตามการใช้งานและจำกัดจำนวน
   - บังคับใช้วันหมดอายุ
   - การตรวจสอบคุณสมบัติ

---

### ฟีเจอร์สำหรับแอดมินและการจัดการ

6. **[แดชบอร์ดแอดมินและการวิเคราะห์ (พร้อม GA4)](./admin-dashboard.md)**
   - ตัวชี้วัดประสิทธิภาพหลัก (KPIs)
   - สถิติการจองและติดตามรายได้
   - การรวมระบบ Google Analytics 4 (ฝั่งไคลเอนต์ + เซิร์ฟเวอร์)
   - การแสดงผลข้อมูลด้วยกราฟ (Recharts)
   - ติดตามเหตุการณ์แบบกำหนดเอง (การจอง, แชทบอท)
   - เมตริกและการติดตามการแปลงแบบเรียลไทม์
   - การวิเคราะห์ปริมาณการเข้าชมและพฤติกรรมผู้ใช้

7. **[สถาปัตยกรรมฐานข้อมูลและ Supabase](./database-supabase.md)**
   - โครงสร้างฐานข้อมูล PostgreSQL
   - การรวมระบบ Supabase Auth
   - Storage buckets สำหรับรูปภาพ
   - การค้นหาด้วย Vector ใช้ pgvector
   - การสมัครรับข้อมูลแบบเรียลไทม์
   - Row-level security (RLS)

8. **[ระบบจัดการข้อมูลโรงแรม](./hotel-info-management.md)**
   - รายละเอียดโรงแรมแบบรวมศูนย์
   - อัปโหลดและจัดการโลโก้
   - ตัวแก้ไขคำอธิบาย
   - สถานะส่วนกลางผ่าน React Context

9. **[ระบบจัดการโปรไฟล์ลูกค้า](./customer-profile.md)**
   - อัปโหลดรูปโปรไฟล์
   - แก้ไขข้อมูลส่วนตัว
   - ความปลอดภัยของบัญชี
   - การตรวจสอบข้อมูล

## 🏗️ ภาพรวมสถาปัตยกรรมระบบ

### เทคโนโลยีที่ใช้

**ฝั่ง Frontend:**
- **Framework:** Next.js 15 (React 19)
- **Styling:** Tailwind CSS 4
- **State Management:** React Context API
- **Form Handling:** React Hook Form + Zod validation
- **UI Components:** Radix UI, shadcn/ui
- **Charts:** Recharts
- **Animations:** Motion (Framer Motion)

**ฝั่ง Backend:**
- **BaaS:** Supabase (PostgreSQL, Auth, Storage, Realtime)
- **API:** Next.js API Routes (serverless)
- **Authentication:** Supabase Auth (JWT tokens)
- **File Storage:** Supabase Storage

**AI และการวิเคราะห์:**
- **LLM:** Google Vertex AI (Gemini 2.5 Flash)
- **Embeddings:** OpenAI (text-embedding-3-small)
- **Vector Search:** Supabase pgvector
- **Analytics:** Google Analytics 4 + GA Data API

**การชำระเงิน:**
- **Payment Gateway:** Stripe (โหมดทดสอบ)
- **วิธีการรองรับ:** บัตรเครดิต, เงินสด

**DevOps:**
- **Hosting:** Vercel (Next.js deployment)
- **Database:** Supabase (managed PostgreSQL)
- **Version Control:** Git

## 📊 ภาพรวมโครงสร้างฐานข้อมูล

### ตารางหลัก

**ผู้ใช้และโปรไฟล์:**
- `auth.users` (จัดการโดย Supabase)
- `profiles` (ข้อมูลผู้ใช้แบบกำหนดเอง)

**การดำเนินงานโรงแรม:**
- `rooms` (ห้องพักที่สามารถจองได้)
- `room_types` (เทมเพลตห้องพัก)
- `bookings` (การจองห้อง)
- `payments` (บันทึกการชำระเงิน)
- `promo_codes` (โค้ดส่วนลด)

**ระบบแชทบอท:**
- `chatbot_sessions` (เซสชันการแชท)
- `chatbot_messages` (ข้อความแชท)
- `chatbot_faqs` (ฐานความรู้ FAQ)
- `chatbot_faq_aliases` (การใช้คำพ้องความหมาย)
- `chatbot_contexts` (บริบทเพิ่มเติม)
- `chatbot_tickets` (ตั๋วสนับสนุน)

**แอดมิน:**
- `hotel_info` (รายละเอียดโรงแรม)
- `ga4_stats` (แคชข้อมูลการวิเคราะห์)

## 🚀 คู่มือเริ่มต้นสำหรับนักพัฒนา

### ข้อกำหนดเบื้องต้น
- Node.js 18.17.0+
- npm 9.0.0+
- บัญชี Supabase
- บัญชี Google Cloud (สำหรับ Vertex AI)
- คีย์ OpenAI API
- บัญชี Stripe (โหมดทดสอบ)

### ตัวแปร Environment ที่ต้องมี

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=

# OpenAI
OPENAI_API_KEY=

# Google Vertex AI
GCLOUD_PROJECT_ID=
GCLOUD_LOCATION=
GOOGLE_APPLICATION_CREDENTIALS_VERTEX_JSON=

# Google Analytics
NEXT_PUBLIC_GA_MEASUREMENT_ID=
GA4_PROPERTY_ID=

# Stripe
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=
STRIPE_SECRET_KEY=
```

### การติดตั้ง

```bash
# ติดตั้ง dependencies
npm install

# รันเซิร์ฟเวอร์สำหรับพัฒนา
npm run dev

# Build สำหรับ production
npm run build

# เริ่มเซิร์ฟเวอร์ production
npm start
```

## 🔗 จุดเชื่อมต่อหลัก

### การยืนยันตัวตน → ฟีเจอร์ที่มีการป้องกัน
- แดชบอร์ดแอดมินต้องการ `role = 'admin'`
- การจองต้องมีลูกค้าที่ยืนยันตัวตนแล้ว
- การจัดการโปรไฟล์ต้องเข้าสู่ระบบ

### แชทบอท → แชทสด → ตั๋วสนับสนุน
- แชทบอทสามารถส่งต่อไปยังแชทสด
- ตั๋วเชื่อมโยงเซสชันแชทกับเจ้าหน้าที่แอดมิน
- บอทหยุดทำงานระหว่างแชทสด

### การจอง → การชำระเงิน → ความพร้อมของห้อง
- การจองตรวจสอบห้องว่าง
- การชำระเงินยืนยันการจอง
- อัปเดตสถานะห้องเมื่อเช็คอิน/เช็คเอาท์

### แดशบอร์ดแอดมิน → Google Analytics
- ดึงข้อมูล GA4 ผ่าน Data API
- แสดงเมตริกในกราฟ
- แคชข้อมูลเพื่อประสิทธิภาพ
