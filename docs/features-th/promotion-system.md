# ฟีเจอร์: ระบบโปรโมชันและโค้ดส่วนลด

## 1. ภาพรวม

ระบบโปรโมชันจัดการโค้ดส่วนลด (promo codes) ที่ลูกค้าสามารถใช้กับการจองเพื่อรับส่วนลด รองรับทั้งส่วนลดแบบจำนวนเงินคงที่และแบบเปอร์เซ็นต์ พร้อมฟีเจอร์เช่น วันหมดอายุ ขีดจำกัดการใช้ และการควบคุมสถานะทำงาน/ไม่ทำงาน

**วัตถุประสงค์:**
- สร้างและจัดการโค้ดส่วนลดโปรโมชัน
- ใช้ส่วนลดกับการจองระหว่างชำระเงิน
- ติดตามการใช้งานโค้ดส่วนลดและบังคับใช้ขีดจำกัด
- รองรับทั้งส่วนลดจำนวนเงินคงที่และเปอร์เซ็นต์
- ควบคุมความถูกต้องของโค้ดผ่านวันหมดอายุ
- ติดตามประสิทธิภาพโค้ดส่วนลด

**ฟีเจอร์หลัก:**
- **ส่วนลดสองประเภท**: จำนวนเงินคงที่ (฿500 off) หรือเปอร์เซ็นต์ (20% off)
- **การติดตามการใช้งาน**: นับจำนวนครั้งที่ใช้โค้ด
- **ขีดจำกัดการใช้**: cap max_uses เป็นตัวเลือก (0 = ไม่จำกัด)
- **วันหมดอายุ**: ยกเลิกอัตโนมัติหลัง expires_at
- **สถานะทำงาน/ไม่ทำงาน**: แอดมินสามารถเปิด/ปิดโค้ด
- **ไม่สนใจตัวพิมพ์**: โค้ดเก็บเป็นตัวพิมพ์ใหญ่ การค้นหาไม่สนใจตัวพิมพ์
- **การตรวจสอบคุณสมบัติ**: ตรวจสอบโค้ดก่อนใช้

## 2. สถาปัตยกรรม / ลำดับการทำงาน

### ลำดับการใช้โค้ดส่วนลด (ลูกค้า)
```
ลูกค้าป้อนโค้ดส่วนลดเมื่อชำระเงิน → ตรวจสอบโค้ดส่วนลด
  → ขั้นตอนที่ 1: ตรวจสอบว่าโค้ดมีอยู่ (ไม่สนใจตัวพิมพ์)
  → ขั้นตอนที่ 2: ตรวจสอบสถานะทำงาน (is_active = true)
  → ขั้นตอนที่ 3: ตรวจสอบหมดอายุ (expires_at > now)
  → ขั้นตอนที่ 4: ตรวจสอบขีดจำกัดการใช้ (used_count < max_uses หรือ max_uses = 0)
  → ขั้นตอนที่ 5: คำนวณส่วนลด
    ├─ ถ้า discount_amount → ส่วนลด = min(discount_amount, total)
    └─ ถ้า discount_percent → ส่วนลด = (total * discount_percent) / 100
  → ขั้นตอนที่ 6: ใช้กับยอดรวมการจอง
    └─ new_total = original_total - discount
  → ขั้นตอนที่ 7: เพิ่ม used_count
  → สำเร็จ → แสดงราคาหลังส่วนลด
```

### ลำดับการสร้างโค้ดส่วนลด (แอดมิน)
```
แอดมินสร้างโค้ดส่วนลด → กรอกฟอร์ม:
  ├─ code (เฉพาะ, ตัวพิมพ์ใหญ่)
  ├─ description (ตัวเลือก)
  ├─ discount_amount หรือ discount_percent (ต้องมีอันใดอันหนึ่ง)
  ├─ expires_at (วันที่)
  ├─ max_uses (0 = ไม่จำกัด)
  └─ is_active (ค่าเริ่มต้น true)
→ ตรวจสอบความเฉพาะ
→ แทรกลงในตาราง promo_codes
→ สำเร็จ → เปลี่ยนเส้นทางไปรายการโปรโมชัน
```

### ลำดับการตรวจสอบโค้ดส่วนลด
```
validatePromotionCode(code, roomId, checkIn, checkOut):
  // 1. ตรวจสอบการมีอยู่
  promo = SELECT * FROM promo_codes WHERE code = UPPER(code)
  ถ้าไม่พบ → ส่งคืน { isValid: false, error: 'โค้ดไม่ถูกต้อง' }
  
  // 2. ตรวจสอบสถานะทำงาน
  ถ้า !promo.is_active → ส่งคืน { isValid: false, error: 'โค้ดไม่ทำงาน' }
  
  // 3. ตรวจสอบหมดอายุ
  ถ้า NOW() > promo.expires_at → ส่งคืน { isValid: false, error: 'โค้ดหมดอายุ' }
  
  // 4. ตรวจสอบขีดจำกัดการใช้
  ถ้า promo.max_uses > 0 และ promo.used_count >= promo.max_uses:
    ส่งคืน { isValid: false, error: 'ถึงขีดจำกัดการใช้' }
  
  // 5. ส่งคืนโปรโมชันที่ถูกต้อง
  ส่งคืน { isValid: true, promotionCode: promo }
```

## 4. ตรรกะหลัก

### 4.1 การคำนวณส่วนลด

```typescript
function calculateDiscount(baseAmount: number, promotionCode: PromotionCode):
  ถ้า !promotionCode.isValid → ส่งคืน 0
  
  // ลำดับความสำคัญ 1: จำนวนเงินส่วนลดคงที่ (ที่ต้องการ)
  ถ้า promotionCode.discount_amount และ promotionCode.discount_amount > 0:
    // ไม่สามารถลดมากกว่ายอดรวม
    ส่งคืน Math.min(promotionCode.discount_amount, baseAmount)
  
  // ลำดับความสำคัญ 2: ส่วนลดเปอร์เซ็นต์ (สำรอง)
  ถ้า promotionCode.discount_percent และ promotionCode.discount_percent > 0:
    ส่งคืน (baseAmount * promotionCode.discount_percent) / 100
  
  // ลำดับความสำคัญ 3: Legacy discountType (backward compatibility)
  ถ้า promotionCode.discountType === 'fixed':
    ส่งคืน Math.min(promotionCode.discount, baseAmount)
  ถ้า promotionCode.discountType === 'percentage':
    ส่งคืน (baseAmount * promotionCode.discount) / 100
  
  ส่งคืน 0
```

**ตัวอย่างการคำนวณ:**
```
สถานการณ์ที่ 1: ส่วนลดจำนวนเงินคงที่
  ยอดรวมเดิม: ฿10,000
  โปรโมชัน: discount_amount = 1500
  ส่วนลด: ฿1,500
  ยอดรวมสุดท้าย: ฿8,500

สถานการณ์ที่ 2: ส่วนลดเปอร์เซ็นต์
  ยอดรวมเดิม: ฿10,000
  โปรโมชัน: discount_percent = 20
  ส่วนลด: ฿2,000 (20% ของ 10,000)
  ยอดรวมสุดท้าย: ฿8,000

สถานการณ์ที่ 3: ส่วนลดคงที่เกินยอดรวม
  ยอดรวมเดิม: ฿500
  โปรโมชัน: discount_amount = 1000
  ส่วนลด: ฿500 (จำกัดที่ยอดรวม)
  ยอดรวมสุดท้าย: ฿0
```

### 4.2 การติดตามการใช้งาน

```typescript
async incrementPromoUsedCount(promoCode: string):
  // 1. รับ used_count ปัจจุบัน
  promo = SELECT used_count FROM promo_codes WHERE code = UPPER(promoCode)
  
  // 2. เพิ่ม used_count
  อัปเดต promo_codes SET 
    used_count = promo.used_count + 1,
    updated_at = NOW()
  WHERE code = UPPER(promoCode)
  
  Log: "เพิ่ม used_count สำหรับ: {promoCode}"
```

**เรียกหลังจาก:**
- การจองสำเร็จพร้อมโค้ดส่วนลด
- การชำระเงินเสร็จสิ้น (ไม่เรียกเมื่อชำระเงินล้มเหลว)

```typescript
async checkPromotionEligibility(userId: string, promotionCode: string):
```

## 5. โมเดลข้อมูล / โครงสร้างฐานข้อมูล

### ตาราง: `promo_codes`
```sql
CREATE TABLE promo_codes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  
  -- รายละเอียดโค้ด
  code TEXT UNIQUE NOT NULL,              -- โค้ดตัวพิมพ์ใหญ่ (เช่น "SUMMER20")
  description TEXT,                       -- คำอธิบายที่มนุษย์อ่านได้
  
  -- การกำหนดค่าส่วนลด (ต้องตั้งค่าหนึ่งอย่าง)
  discount_amount DECIMAL(10,2),          -- จำนวนเงินคงที่ (เช่น 500 = ฿500 off)
  discount_percent DECIMAL(5,2),          -- เปอร์เซ็นต์ (เช่น 20 = 20% off)
  
  -- การควบคุมความถูกต้อง
  is_active BOOLEAN DEFAULT true,         -- แอดมินสามารถเปิด/ปิด
  expires_at TIMESTAMPTZ NOT NULL,        -- วันยกเลิกอัตโนมัติ
  
  -- การติดตามการใช้งาน
  max_uses INTEGER DEFAULT 0,             -- 0 = ไม่จำกัด, >0 = cap การใช้งาน
  used_count INTEGER DEFAULT 0,           -- ใช้ไปกี่ครั้ง
  
  -- Timestamps
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  
  -- Constraints
  CONSTRAINT valid_discount CHECK (
    (discount_amount IS NOT NULL AND discount_amount > 0) OR
    (discount_percent IS NOT NULL AND discount_percent > 0 AND discount_percent <= 100)
  ),
  CONSTRAINT valid_max_uses CHECK (max_uses >= 0),
  CONSTRAINT valid_used_count CHECK (used_count >= 0),
  
  -- Indexes
  INDEX idx_promo_code ON promo_codes(UPPER(code)),
  INDEX idx_active_promos ON promo_codes(is_active, expires_at)
);
```

### สถานะวงจรชีวิตของโค้ดส่วนลด
```
┌─────────┐    แอดมิน    ┌────────┐    หมดอายุ    ┌─────────┐
│ สร้าง │ ────────────> │ ทำงาน │ ───────────> │ หมดอายุ │
└─────────┘    เปิดใช้     └────────┘  (อัตโนมัติ)  └─────────┘
                               │
                               │ แอดมินปิด
                               ↓
                          ┌──────────┐
                          │ ไม่ทำงาน │
                          └──────────┘
```

## 6. กรณีพิเศษ / ข้อจำกัด / TODO

### กรณีพิเศษที่จัดการแล้ว
1. **จำกัดส่วนลด**: ส่วนลดจำนวนเงินคงที่ไม่สามารถเกินยอดรวมการจอง
2. **ไม่สนใจตัวพิมพ์**: โค้ดจับคู่ด้วยการแปลง UPPER()
3. **ขีดจำกัดการใช้**: บังคับใช้ cap max_uses (ไม่จำกัดถ้าเป็น 0)
4. **การบังคับใช้หมดอายุ**: ปฏิเสธโค้ดที่หมดอายุอัตโนมัติ
5. **การตรวจสอบสถานะทำงาน**: เฉพาะโค้ดที่ทำงานเท่านั้นที่ถูกต้อง
6. **การตรวจสอบเปอร์เซ็นต์**: ตรวจสอบเปอร์เซ็นต์ ≤ 100%
7. **หนึ่งโปรโมชันต่อการจอง**: ไม่สามารถซ้อนโค้ดส่วนลดหลายตัว

### ข้อจำกัดปัจจุบัน
1. **ไม่มีการซ้อนโค้ด**: ไม่สามารถใช้โค้ดส่วนลดหลายตัวต่อการจองเดียว
2. **ไม่มีโค้ดเฉพาะผู้ใช้**: โค้ดทั้งหมดเป็นสาธารณะ (ไม่มีโค้ดส่วนบุคคล)
3. **ไม่มีการสั่งซื้อขั้นต่ำ**: ไม่มีข้อกำหนดจำนวนเงินจองขั้นต่ำ
4. **ไม่มีข้อจำกัดประเภทห้อง**: โค้ดใช้ได้กับห้องทุกประเภท
5. **ไม่มีข้อจำกัดวันที่**: ไม่สามารถจำกัดวันที่จองเฉพาะ
6. **ไม่มีสำหรับผู้ใช้ครั้งแรกเท่านั้น**: ไม่มีการตรวจสอบลูกค้าใหม่ vs เก่า
7. **ไม่มีหมวดหมู่โปรโมชัน**: ไม่มีการจัดกลุ่ม/จำแนกโปรโมชัน
8. **ไม่มีการใช้อัตโนมัติ**: โปรโมชันที่ดีที่สุดไม่ถูกใช้อัตโนมัติ

### ปัญหาที่ทราบ
- **ระยะเวลาเพิ่ม Used Count**: เพิ่มขึ้นแม้ว่าการชำระเงินล้มเหลว (ควรเพิ่มเฉพาะเมื่อชำระเงินสำเร็จ)
- **ไม่มีประวัติโปรโมชัน**: ไม่สามารถดูว่าใครใช้โค้ดอะไร
- **ไม่มีการตรวจจับการฉ้อโกง**: ไม่มีการป้องกันการใช้โค้ดในทางที่ผิด
- **การใช้พร้อมกัน**: Race condition เมื่อผู้ใช้หลายคนใช้โค้ดใกล้ขีดจำกัด
- **ไม่มีผลกระทบการลบโปรโมชัน**: การลบโปรโมชันจากการจองไม่ลด used_count
- **โค้ดหมดอายุไม่ถูกเก็บถาวร**: โค้ดหมดอายุยังคงอยู่ในตารางทำงาน (ไม่มีการเก็บถาวร)
